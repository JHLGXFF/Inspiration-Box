## Why

目前的灵感收集箱只支持文字和 URL 链接形式的灵感记录，但灵感往往以多种形式出现——一张照片、一段视频、一个截图。为了让用户能够更全面地捕捉和保存灵感，需要扩展系统以支持图片和视频等多媒体内容的上传、存储和展示。

## What Changes

- 扩展灵感记录功能，支持上传图片和视频文件
- 在后端添加文件上传处理、存储和服务能力
- 在前端添加多媒体文件选择、预览和上传界面
- 更新数据模型以支持多媒体附件的元数据
- 在卡片流展示中整合多媒体内容的预览和播放

## Capabilities

### New Capabilities

- `media-upload`: 多媒体文件上传功能，包括图片和视频的选择、验证、上传处理
- `media-storage`: 多媒体文件存储和管理，包括文件保存、命名、路径管理
- `media-display`: 多媒体内容展示功能，包括缩略图生成、图片预览、视频播放

## Impact

**后端影响**：
- 数据模型：需要为 Memo 模型添加多媒体附件字段（文件路径、文件类型、文件大小等）
- API 端点：需要添加文件上传端点，更新现有的 memo 创建/获取端点
- 文件系统：需要创建媒体文件存储目录结构
- 依赖项：可能需要添加文件处理库（如图片压缩、视频处理）

**前端影响**：
- UI 组件：需要添加文件选择器、拖拽上传区域、媒体预览组件
- 状态管理：需要处理文件上传进度、多媒体数据加载
- 展示组件：卡片流需要支持显示图片和视频

**存储影响**：
- 需要考虑存储空间管理和文件大小限制
- 需要确保文件路径的安全性和访问控制
