# Data Model

## Memo
- id: integer, primary key
- content: string, required, non-empty (text or URL)
- created_at: datetime, required

## Tag
- id: integer, primary key
- name: string, required, unique, trimmed lowercase

## MemoTag
- memo_id: integer, foreign key to Memo
- tag_id: integer, foreign key to Tag
- primary key: (memo_id, tag_id)

## Relationships
- Memo 1..n MemoTag
- Tag 1..n MemoTag
- Memo n..n Tag (through MemoTag)

## Validation Rules
- content must be non-empty; if content parses as URL, store as-is.
- tags are optional; if provided, each tag name is 1-30 chars after trim.
- created_at is generated by the backend at insert time.

## State Transitions
- None. Memos are immutable after creation in this MVP.
